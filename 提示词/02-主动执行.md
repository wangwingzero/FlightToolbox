# 主动执行指令

## 指令内容

```
Take initiative and make bold changes with minimal confirmation. You need to carefully read all the code that you are supposed to read before giving an explanation, without any guesswork. Follow plan.md for feature development: When developing new features, strictly follow the steps listed in the plan.md.
```

## 指令含义

**核心原则：果断执行，减少犹豫**

这个指令要求AI在详细计划制定完成后，要主动、果断地执行修改，避免过度的确认和犹豫。这是执行阶段的核心指导原则。

## 适用场景

### 1. 计划确定后的实施阶段
- 按照既定计划执行代码修改
- 实施已验证的技术方案
- 执行标准化的开发流程

### 2. 明确的问题修复
- 已定位的Bug修复
- 性能优化实施
- 安全漏洞修补

### 3. 标准化操作
- 代码重构执行
- 配置文件更新
- 依赖库升级

## 执行原则

### 前提条件

1. **详细计划已制定**
   - 必须先完成"详细实施计划"阶段
   - 所有代码文件已仔细审查
   - 实施步骤清晰明确

2. **风险评估完成**
   - 潜在风险已识别
   - 回滚方案已准备
   - 测试验证方法已确定

### 执行要求

1. **主动性**
   - 不等待额外确认就开始执行
   - 遇到小问题时自主解决
   - 主动优化代码质量

2. **果断性**
   - 按计划坚决执行
   - 不因小的不确定性而停滞
   - 相信前期分析的准确性

3. **完整性**
   - 一次性完成所有相关修改
   - 确保修改的一致性
   - 避免半成品状态

## 与第一阶段的关系

### 互补关系
- **第一阶段**：谨慎分析，详细规划
- **第二阶段**：果断执行，快速实施

### 平衡原则
- 规划阶段要足够谨慎
- 执行阶段要足够果断
- 避免在执行阶段重新规划
