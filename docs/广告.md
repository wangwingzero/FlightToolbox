# 总则

1.广告不需要加入复杂的样式，尽量简洁，只使用基础的广告组件，不建议自定义样式。这样可以减少不必要的代码量。
2.广告位ID禁止使用这个页面以外的广告位ID，否则会导致广告加载失败。
3.总体原则是在不同的功能之间增加广告，底部一般可以增加一个广告。
4.正常情况下优先使用横幅广告，只有在一个页面上连续过多横幅广告的情况下才使用格子广告替换。

# 横幅广告

## 示例代码

// wxml文件
`<ad-custom unit-id="adunit-4e68875624a88762" bindload="adLoad" binderror="adError" bindclose="adClose"></ad-custom>`

// js文件，广告事件监听
Page({
  adLoad() {
    console.log('横幅广告加载成功')
  },
  adError(err) {
    console.error('横幅广告加载失败', err)
  },
  adClose() {
    console.log('横幅广告关闭')
  },
})

# 格子广告（原生模板广告）

## 使用场景

⚠️ **仅在特殊情况下使用**：当一个页面上连续过多横幅广告时，使用格子广告来替换，避免广告与音频播放等功能冲突。

正常情况下应优先使用横幅广告，格子广告仅作为特殊情况的替代方案。

## 示例代码

// wxml文件
`<ad-custom unit-id="adunit-735d7d24032d4ca8" bindload="adLoad" binderror="adError" bindclose="adClose"></ad-custom>`

// js文件，广告事件监听
Page({
  adLoad() {
    console.log('原生模板广告加载成功')
  },
  adError(err) {
    console.error('原生模板广告加载失败', err)
  },
  adClose() {
    console.log('原生模板广告关闭')
  },
})

## 格子广告位ID

adunit-735d7d24032d4ca8

### 格子广告注意要点

1. 仅在页面横幅广告过多导致冲突时使用
2. 统一使用同一个广告位ID
3. 格子广告形式更灵活，兼容性更好
4. 适用于需要避免与音频播放等功能冲突的场景

## 应用实例

**航线录音页面**：由于页面包含音频播放功能，且有7个广告位，使用格子广告避免与音频播放冲突。

# 激励广告

## 激励广告示例代码

// 若在开发者工具中无法预览广告，请切换开发者工具中的基础库版本
// 在页面中定义激励视频广告
let videoAd = null

// 在页面onLoad回调事件中创建激励视频广告实例
if (wx.createRewardedVideoAd) {
  videoAd = wx.createRewardedVideoAd({
    adUnitId: 'adunit-316c5630d7a1f9ef'
  })
  videoAd.onLoad(() => {})
  videoAd.onError((err) => {
    console.error('激励视频光告加载失败', err)
  })
  videoAd.onClose((res) => {})
}

// 用户触发广告后，显示激励视频广告
if (videoAd) {
  videoAd.show().catch(() => {
    // 失败重试
    videoAd.load()
      .then(() => videoAd.show())
      .catch(err => {
        console.error('激励视频 广告显示失败', err)
      })
  })
}

## 激励广告广告位ID

adunit-316c5630d7a1f9ef

### 激励广告注意要点

激励广告就一个，并且不要增加，也不要改变相关代码。
