# 等待程序进入方式训练

## 功能说明

本模块提供两种模式：

### 1. 练习模式（默认）
- 系统随机生成航向、入航航道和转弯方向
- 用户选择正确的进入方式（直接进入、偏置进入、平行进入）
- 显示答题结果和统计数据
- 播放进入动画

### 2. 实际使用模式
- 飞行员输入管制员给出的信息：
  - 当前航向 (HDG)
  - 入航航道 (IB CRS)
  - 等待转弯方向（左转/右转）
- 系统计算并显示推荐的进入方式
- 显示相对角度
- 如果在边界区域，提示可使用其他进入方式
- 可查看进入动画

## 进入方式判断规则

根据中国民航规章，等待程序分为三个扇区：

### 右转等待（标准）
- **第3扇区（直接进入）**：180° 范围
  - 相对角度 0-110° 和 290-360°
  - 到达定位点后，右转弯遵循等待航线飞行

- **第2扇区（偏置进入）**：70° 范围
  - 相对角度 110-180°
  - 到达定位点后，在等待航线一侧与入航航迹的反向成 30° 的航迹飞行
  - 右转弯切入等待入航航迹

- **第1扇区（平行进入）**：110° 范围
  - 相对角度 180-290°
  - 到达定位点后，向左转至出航航向飞行
  - 向左转向等待一侧，切入入航航迹或回至定位点

### 左转等待
扇区划分镜像翻转，偏置扇区（70°）位于入航航迹左侧。

### 边界容差
在扇区边界 ±5° 范围内，可以使用相邻扇区的进入方式。

## 使用示例

### 实际使用场景
管制员指令："保持等待，入航航道 090，右转等待"

飞行员操作：
1. 切换到"实际使用"模式
2. 输入当前航向，例如 270
3. 输入入航航道 090
4. 选择右转
5. 点击"计算进入方式"
6. 系统显示：平行进入（Parallel）
7. 可点击"查看动画"观看进入轨迹

## 技术实现

- 使用 Canvas 2D API 绘制罗盘和等待航线
- 使用 `aviation-math.js` 计算进入类型
- 支持动画播放和重播
- ES5 语法兼容微信小程序
