## 📝 版本更新步骤

> **重要约定**：每次使用本文件编写/更新某个版本的说明时，都默认已经把所有和"版本信息"相关的代码一并更新完毕（包括首页 `onVersionTap` 弹窗文案、首页版本显示、以及其他页面中出现的版本号）。

### 💡 快速获取更新内容

从 git 提交记录中获取本版本的改动，运行：

```bash
git log --oneline -30
```

查看类似 `chore: 2.14.0-1修订版...` 的提交信息，汇总各次提交的改动点即可。

4. **同步修改** `miniprogram/pages/home/index.wxml` 中"版本信息"单元格的 `value` 属性（约第241行，`<van-cell title="版本信息" value="vX.X.X" .../>`）
5. 检查其他页面是否有版本号需要同步更新
6. 在需要做版本对比的说明中，用一行 `@[d:\\FlightToolbox\\更新说明\\版本更新说明_v2.11.0.md]` 标明参照版本（示例，实际使用时替换为上一版或需要对比的版本）

## 🎯 文案风格规则（一定要口语化）

- **面向飞行员同事**：假装是在和机长/副驾驶聊天说明“这次更新改了啥”，不要出现“接口、脚本、npm”等技术词
- **讲用户感受，不讲实现细节**：多说“对你有什么用”，少讲“底层逻辑怎么实现”
- **简短好读**：控制在约 200 字内，分行/分点，避免一整段大段文字
- **语气友好、自然**：可以适当用表情符号（✈️、😊 等），但不要过多
